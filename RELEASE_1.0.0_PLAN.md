### Bugs uncovered by T2 (block 1.0.0; fix before next phase)

- [x] **T2.1. SOUNDNESS: compiled-expr parser fails on nested `[r EXCEPT !.f = v]`.** Closed — `try_parse_except` now scans for `EXCEPT` only at top-level bracket depth via the new `find_top_level_except` helper. `[[r EXCEPT !.a = 0] EXCEPT !.b = 0]` now compiles to `FuncExcept(FuncExcept(r, [a→0]), [b→0])` and evaluates to `[a |-> 0, b |-> 0]`. See log entry `T2.1 + T2.2 — nested / record-field EXCEPT`.
- [x] **T2.2. SOUNDNESS: compiled-expr parser fails on EXCEPT inside a record-literal field value.** Closed by the same fix as T2.1 — the inner EXCEPT inside `[a |-> 0, b |-> ([r EXCEPT !.a = 0]).a]` is no longer matched by `try_parse_except`'s outer scan, so the record literal path correctly takes over. Evaluates to `[a |-> 0, b |-> 0]` matching the interpreter. See log entry `T2.1 + T2.2`.
- [x] **T2.3. SOUNDNESS: interpreter type-errors on `<range> \subseteq <set>` (and `\intersect`).** Closed — `..` (TLA+ precedence 9-9) was lumped into the relational/comparison tier in the interpreter (`split_top_level_comparison`) and placed *above* the set-op block in the compiler cascade. Both directions silently misparsed `n..m \subseteq S` and `S \union n..m` shapes. Fix: removed `..` from comparison patterns and added a dedicated `split_top_level_range` helper in the interpreter; reordered the compiler cascade so set ops (8-8) come before `..` (9-9) come before additive (10-10). 8 regression tests, 13/13 diff harness pass, proptest harness now green across 9 seeds. See log entry `T2.3 — \`..\` precedence vs set ops`.

## Phase 2 — High-leverage performance

- [x] **T5. Symbolic Init enumeration.** Z3-backed enumeration for filtered record sets that currently brute-force. Behind `symbolic-init` feature flag (default off; requires libz3-dev + clang). Module: `src/tla/symbolic_init.rs`. Wired into `eval_set_expression` immediately before brute-force fallback. Translator handles `/\`, `\/`, `~`, `=>`, `<=>`, `=`, `#`, `<`, `<=`, `>=`, `>`, `+`, `-`, `*`, `\in {literal,...}`, `\in lo..hi`, field access `tup.f`, and quantifiers expanded over finite literal domains. Constant folding via the outer evaluator handles arbitrary subexpressions that don't reference the bound record variable. Domain identifier resolution: when the record-set domain is a name like `Can` rather than a literal `[f1: D1, ...]`, the new code resolves it through `ctx.definitions` so the fast path also fires. Benchmark on `TightCan` (5-field record set with sortedness + sum constraint): N=15 brute-force 14.71s → symbolic 1.40s (10.5x); N=20 brute-force 58.42s → symbolic 1.41s (41x); N=40 brute-force fails on eval budget → symbolic 1.6s. Correctness gate (proptest equivalence + brute-force-on-small) green at 64 random cases. 671 tests pass with the feature on (vs 663 without; +8 = 6 unit + 2 integration). Einstein still falls back gracefully (its Init shape is `var \in {p \in PermutationSet : pred(p)}` over sequences — outside the supported subset). See `### T5` log entry for details and follow-up T5.N entries.

### Follow-ups (parked)

- [ ] **T5.1. Sequence-set Init.** Einstein-shape: `var \in {seq \in [Domain -> Range] : pred(seq)}` plus the function-set composition `{ FunAsSeq(p, n, n) : p \in {p \in [...] : ...} }`. Requires extending the translator to encode sequences as Z3 arrays or per-position Int vars and handle `FunAsSeq` symbolically.
- [ ] **T5.2. Permutation symmetry.** Einstein's `Permutation(S)` enumerates all permutations of a 5-element set, then filters. With AllSAT-style "block-by-projection" or by treating the permutation as a constraint problem (Z3's distinct), we can collapse symmetric solutions before enumeration.
- [ ] **T5.3. Replace block-and-resolve with projection-based all-SAT.** When the solution count is large but each solution is cheap to extract, the per-solution SMT call is the bottleneck. Either (a) batch-extract via Z3's projection enumeration API, or (b) detect when `pred` is a near-tautology and skip symbolic for that case.
- [x] **T6. Cross-node distributed work stealing.** Done — TCP-based steal protocol implemented (`spawn_steal_trigger_task` + steal-victim threshold + peer-down cooldown + termination consensus extension). 5 unit tests + 3 integration tests including the dead-peer failure mode. **Wider scope discovered**: surfaced and fixed three pre-existing termination-detection bugs in v0.3.0 distributed mode that prevented any multi-node cluster from converging. Wall-time speedup on the synthetic 2-node benchmark was negative (cluster mode actually slower than independent on `RunCounterGrid` because each node's local FP store leads to redundant exploration of the same state space) — see `RELEASE_1.0.0_LOG.md` for the honest read. The protocol is correct and fires; the deeper "make distributed actually faster" question requires global FP partitioning, which is out of scope for 1.0.0 per the brief. See log for commit chain.
- [x] **T7. Partial-order reduction (POR).** Done — stubborn-set POR with static read/write dependency analysis. Module `src/tla/por.rs` (new). Behind opt-in `--por` CLI flag, default off. Safety only — automatically rejected when fairness/liveness present (rejection at `enable_por()` with explanatory error). Benchmark on `PorBenchProcessGrid` (4 independent processes, MAX=4): full=625 states, POR=17, **36.8x reduction**, **17.9x wall-time speedup**. PorBenchPipeline (shared `pending` var) shows the no-reduction worst case — POR overhead nets ~0.6x speedup, accepted with documentation. PorTwoCounters: 2.3x reduction, 1.6x speedup. 686 tests pass (+15: 9 unit + 5 correctness + 1 benchmark). Correctness gates: POR reachable set ⊆ full set on every fixture; safety-violation parity confirmed (PorViolation reaches `count >= 3` under both modes); liveness opt-out tested. See `### T7` log entry for details.

### Follow-ups (parked)

- [ ] **T7.1. Batched per-disjunct evaluation.** Today POR calls `evaluate_next_states_swarm(.., &[idx])` once per disjunct, doing N-1 redundant `split_action_disjuncts` passes. A new "evaluate all disjuncts, return Vec<Vec<TlaState>>" API would amortize parsing and shrink the no-reduction overhead seen in PorBenchPipeline.
- [ ] **T7.2. Smarter stubborn-set seed.** Lowest-index seed is deterministic but not optimal. A seed-selection heuristic (e.g., minimum static dependency closure, or "bias toward locally-stable variables") may compress further on real-world specs.
- [ ] **T7.3. POR for liveness via "ignored states" condition.** Adds an extra constraint that no infinite trace can be permanently ignored. Standard but adds complexity; deferred until T10 (liveness scaling).

## Phase 3 — Polish

- [x] **T8. State compression in queue.** Done — zstd-compressed in-memory ring between hot work-stealing deques and disk overflow. Default on, opt-out via `--queue-compression false`. Benchmark (queue micro-bench, 1M items, 8 workers): 13.2x compression ratio, +2% wall time, -68% peak memory. At 250K items: +21% time, -98% memory. Compression only fires when the spill path engages, so quick runs are unaffected. 8 new tests (5 ring unit + 3 integration); 694 total. See `### T8` log entry. Commit `fa378c9`.
- [x] **T9. Trace minimization on violation.** Done — `src/trace_minimize.rs` + `--minimize-trace` CLI flag (default on, 30 s budget). Phase A: earliest-violation truncation pass + BFS shortcut search to fixed point. Phase B: syntactic variable-relevance scan over the invariant body, used by the printer to mark non-referenced variables as "(noise)". Diamond fixture: 9 → 6 states in 375 µs; PorViolation: BFS-optimal trace stays unchanged in 83 µs. 20 new tests, 714 total. Correctness gates preserved across every iteration: trace starts at an initial state, every transition is valid Next, final state still violates the invariant. Follow-ups parked: T9.1 (transitive variable relevance through operator inlining), T9.2 (smarter BFS seed), T9.3 (suffix shortening).
- [x] **T10. Liveness checking scaling.** Done — iterative Tarjan + fingerprint-keyed graph + fast `check_fairness_on_scc_fp` cut the SCC-based fairness post-processing from O(scc_size × tx × constraints) to O(tx × constraints). Bench (LivenessBench N=8: 32 768 states, 143 361 transitions, one giant SCC, 6 WF constraints, 4 cores aarch64): liveness phase wall-time **63.39 s → 115.28 ms (~550x)**, total wall-time **64.92 s → 1.60 s**, peak RSS **2.59 GB → 1.99 GB (-23%)**. N=10 (100 K states) optimized: 484.97 ms phase / 3.34 s wall, baseline killed after 13+ minutes. Module scope: `src/fairness.rs` (iterative Tarjan + new fast-path) + `src/runtime.rs` (post-processing block rewritten). 6 new unit tests (200 K-chain stack-safety, 1K-cycle giant SCC, 4 fast-path semantics) — 720 tests total. T1.3 wrapper-Next regression preserved. See `RELEASE_1.0.0_LOG.md` `### T10` for the run logs and parked follow-ups (T10.1–T10.4).
- [ ] **T11. Long-run soak / chaos.** Run failpoint chaos for hours under random fault injection. Catches accumulation bugs that single-fault tests miss.
- [ ] **T12. Cross-arch CI matrix.** Corpus runs only on aarch64 today. Add x86_64 to CI to catch endianness/atomic-ordering issues.
